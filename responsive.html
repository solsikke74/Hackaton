Responsiv

The viewport

Brukerens synlige område på en webside
Varierer etter skjermstørrelsen.

Kode i meta

meta name="viewport" content="width=device-width, initial-scale=1.0"
Designet blir responsivt etter størrelsen
width=device-width - Designet tilpasses skjermbredden
initial-scale=1.0 - setter utgangspunkt for zooming når siden først lastes i nettleseren

Huskeregler størrelsesforhold

Innhold skal ikke være større enn bredden. Pass på å justere elementer til bredde, så du slipper horisontal skrolling
Innhold skal kunne se bra ut på flere ulike bredder
Bruk % og ikke pixler for best resultat

Grid view

Vi deler inn siden i ulike kolonner
Det gjør det enklere å plassere ulike elementer på websiden.
Et responsivt design har 12 kolonner, som har en total bredde på 100%, og som krymper og forstørres når du forminsker eller forstørrer vinduet

Oppbygning av grid-view

Border box
Bruk box-sizing og sett verdien til border-box. Dette sørger for at padding og border er inkludert i total bredde og høyde til elementet
Dette er koden: * { box-sizing: border-box; }
Du legger inn denne koden i css-filen, og den er global og gjelder for alle nettsider som linkes til dette dokumentet.

Oppsett med 12 kolonner

Vi lager 12 ulike klasser. Bruker div og kaller klassen når vi trenger den.
.col-1 {width: 8.33%;}
.col-2 {width: 16.66%;}
.col-3 {width: 25%;}
.col-4 {width: 33.33%;}
.col-5 {width: 41.66%;}
.col-6 {width: 50%;}
.col-7 {width: 58.33%;}
.col-8 {width: 66.66%;}
.col-9 {width: 75%;}
.col-10 {width: 83.33%;}
.col-11 {width: 91.66%;}
.col-12 {width: 100%;}

Slik kaller vi koden i html. "<"div class="col-9" ">"

Float og kolonner

Hver kolonne skal pakkes inn i en divog flyte til venstre.

Media query

Media query blir brukt for å tilpasse innholdet etter skjermstørrelsen.
Vi bruker ulike "break points", der enkelte deler av designet oppfører seg ulikt, basert på størrelsen
Dette er koden vi legger inn i css.
@media only screen and (max-width: 768px) {
/* For mobile phones: */
[class*="col-"] {
width: 100%;
}
Gå inn på denne siden for å se eksmpel ifbm break points

Mobildesign først

Grunnen er at sidene lastes raskere på små enheter.
I stedet for å designe vinduet til å endre seg når det er lite, bør vi endre det når det blir større.
w3school har 5 "break points"
Her er forslag til kode og hvordan det blir brukt. /* Extra small devices (phones, 600px and down) */ @media only screen and (max-width: 600px) {...}
/* Small devices (portrait tablets and large phones, 600px and up) */ @media only screen and (min-width: 600px) {...}
/* Medium devices (landscape tablets, 768px and up) */ @media only screen and (min-width: 768px) {...}
/* Large devices (laptops/desktops, 992px and up) */ @media only screen and (min-width: 992px) {...}
/* Extra large devices (large laptops and desktops, 1200px and up) */ @media only screen and (min-width: 1200px) {...}
